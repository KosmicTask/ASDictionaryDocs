Class XML import preference

reduce

Description

XML import preferences.

  • Defined in:
    • Preferences Suite

Properties

  • allow transform boolean -- If true, transforms the XML using an XSLT file.
  • class (r/o) type class -- The class descriptor type.
  • create link to XML boolean -- If true, creates a link to the imported XML file. If false, embeds the file.
  • ignore unmatched incoming boolean -- If true, ignores elements that do not match the existing structure. Note: Valid only when import style is merge.
  • ignore whitespace boolean -- If true, leaves existing content in place if the matching XML content contains only whitespace characters such as a carriage return or a tab character. Note: Valid only when import style is merge.
  • import CALS tables boolean -- If true, imports CALS tables as InDesign tables.
  • import style append import / merge import -- The style of incorporating imported XML content into the document.
  • import text into tables boolean -- If true, imports text into tables if tags match placeholder tables and their cells. Note: Valid only when import style is merge.
  • import to selected boolean -- If true, imports into the selected XML element. If false, imports at the root element.
  • object reference (r/o) XML import preference -- An object reference for the object.
  • parent (r/o) reference -- The parent of the XML import preference (a application or document). Can return: application or document.
  • properties record -- A property that allows setting of several properties at the same time.
  • remove unmatched existing boolean -- If true, deletes existing elements or placeholders in the document that do not have matches in the XML file. Note: Valid only when import style is merge.
  • repeat text elements boolean -- If true, repeating text elements inherit the formatting applied to placeholder text. Note: Valid only when import style is merge.
  • transform filename anything -- The name of the XSLT file. Note: Valid when allow transform is true. Can return: alias or string or stylesheet in XML.
  • transform parameters list of anything -- Stylesheet parameters as a list of name/value pairs in the format [[name, value], [name, value], ...]. Can return: Ordered list containing name:string, value:string.
